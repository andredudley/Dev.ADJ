Everything Is What It Isn't: My Unending Beef With JavaScript

March 4, 2026

"Everything is what it is, and everything is what it isn't, and that's why we love it."

I'm not entirely sure who came up with this quote, but I can not think of a more appropriate summation of the existence of JavaScript.

JavaScript is flexible, which can descend into chaos.

JavaScript is forgiving, which gives you more rope to hang yourself.

JavaScript is everywhere, and you cannot escape it.

And that's why we love it. I think. I hardly know myself.

A Bit of Context

Before I implemented 11ty to my blog a little under a month ago, I hadn't done much with JavaScript. I've used it, sure. I even used it to create this very blog. However, I would be lying through my teeth if I told you I was as adept with JS as I am with HTML, CSS, or even backend languages like Python. Especially with the sheer amount of time I was spending in HTML and CSS (in freeCodeCamp might I add. That website is top tier.), it had beenyears since I had truly invested meaningful time into JS. The most I had remembered of it was simple stuff, and how I thought how cool and flexible it was.

Boy, how big of a moron I was.

Enter 11ty.

As I briefly mentioned earlier, I had implemented 11ty into my blog to somewhat automate the process of updating log entries. The process of making new entries into the log is now a somewhat simple six step process:

  1. Create a Markdown file.
  2. Fill the header.
  3. Write whatever below it.
  4. Run the command npm run build in the terminal.
  5. Commit to Repository.
  6. Profit.

Though the process is a bit simplified for the sake of your reading pleasure and me not delving into techno-mumbo-jumbo, this is generally how it goes. But, the process of integrating and setting up 11ty for the log's automation? That was a massive can of worms on its own. I say this because 11ty is written in JS and uses the Node.js framework as its runtime.

The point is: I was, for the first time in a few months at least, completely out of my depth.

I found myself staring into the pit of tutorial purgatory, and did not like in the slightest what was staring back.

What was once a fun little goal for my blog suddenly became a mess of tabs ranging from the 11ty website, to another 11ty tutorial, to a literal tech doc of JS. I got it done, thank goodness, but the feeling that followed wasn't the sense of accomplishment I felt when I used grid to create the layout of this website. Granted, HTML and CSS are markup and stylesheet languages, but any developer or programmer worth their salt knows how tough writing html and css can be.

But, with the depths of tutorial reliance staring into my eyes once again, I decided to take up JavaScript once more. I had it set in my mind that I was never going to rely on these tutorials again.

The Descent

When I was learning how to play fighting games, the path was clear, albeit difficult in the moment:
Lose. Lab. Lose a bit less. Lab some more. Win. Sometimes.

When I was learning HTML and CSS, the feedback was immediate as well. Input text. Input Values. See results. Change values. Break layout. Fix layout.

JavaScript does NOT work like this.

JS finds the sneakiest, most irritating ways to not work. Or, when it feels spontaneous, it will just straight up explode in production because it hasn't gotten the chance to ruin your day yet. I may or may not be speaking from experience.

And the most gutwrenching part? I couldn't just "git gud" at JS the same way I could with HTML, CSS, or other languages like Python. The process of getting good with fighting games could hardly apply here either. The language constantly mutates with new frameworks, and then newer frameworks that people created to solve the problems the older frameworks, and then—just because some developers were in a silly, goofy mood—create another framework. For the love of the game, or something, I don't know.

But, all I could hear is that same, lying voice:

"Maybe I'm just not cut out for this"

That liar.

But, the thing about the voice this time is that the rules of the road, so to speak, have changed. Significantly. Repeatedly. And then they were tossed into a paper shredder and glued back together haphazardly. And, given the fact that JavaScript was made in ten days? It makes sense.

But, What If Jank Is The Point?

I mean, just think about it.

The flexibility of JS makes sense because if there is one way to describe the existence of the World Wide Web; it is flexible. JS is forgiving because the web had to be forgiving to survive the horrendous code being tossed into it. And, at the end of the day, the web won. So, JavaScript won.

Now, JavaScript isn't an inherently bad language in the way that, say, INTERCAL was when it came out in 1972. It is a language that has had to, in the most literal sense, grow up with us and the advent of the World Wide Web. On every device imaginable. Backwards compatible until the collapse of the internet.

You can find a great portion of the history of the World Wide Web in the weirdest, jankiest, most annoying corners of JavaScript. And, in a sense, that's beautiful. Just sucks that I have to deal with it.

The Path Forward

I feel like there is a reason why people switch to Python and never look back. I think about doing the same at times. Then I remember that I like creating things on the web, and the web is built on chaos. So here we are.

And what do I do?

The same I've always done. The same I've done with CSS. The same thing I did with Java. The same thing I did with Python. The same thing I've done with any game or sport I ever picked up in my life.

I start from the bottom. The bare-bones basics. And I suck. A lot. Sometimes intentionally, other times because I genuinely suck. As long as it takes.

I find myself in freeCodeCamp. I find myself in the MDN Docs. I find myself googling the same bug for the fifth time in an afternoon, hoping for an epiphany.

Shoutout to MDN Web Docs, by the way. Not just for JS, but web development in general. HTML, CSS, JS, whatever the language, it felt like the documentation was written by people who were once in my shoes.

No AI slop. No corpo-tech jargon. Just a ton of information with clear explanation with examples and guides.

Every single time I get stuck, I find myself on there. It's practically a web developer's Library of Alexandria—minus the fire.

Use it. Bookmark it. Use it again.

"Everything Is What it Is"

JavaScript is janky. And I keep coming back to it. For some reason.

The opening quote really stuck with me throughout this post. Not because of its profound nature, but because I couldn't help but agree with it at every turn. JavaScript is exactly what it is. And, what it is, is exactly what I need it to be, and also why I have a love-hate relationship with JS.

The same inconclusiveness that drives me up a wall is also what keeps me going. In the midst of my own frustration with the language, all I can think of is—

Man, how cool would it be if I could figure this out.
